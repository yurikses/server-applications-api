# Вторая контрольная работа по FastAPI

## Отчет по выполнению заданий

В данном репозитории реализованы вся контрольная работа №2:

### Задание 3.1: Работа с Pydantic и POST запросами
* Создана Pydantic модель `UserCreate` (в коде `UserRequest`) для валидации данных пользователя (имя, email, возраст, подписка).
* Реализован POST эндпоинт `/create_user`, который принимает данные, присваивает уникальный UUID и сохраняет пользователя.

### Задание 3.2: Обработка продуктов и параметров пути/запроса
* Реализован GET эндпоинт `/product/{product_id}` для получения информации о конкретном товаре по его ID.
* Реализован GET эндпоинт `/products/search` для поиска товаров по ключевому слову (`keyword`), категории (`category`) с ограничением количества результатов (`limit`).
* Маршруты расположены в правильном порядке во избежание конфликтов типизации FastAPI.

### Задание 5.1 и 5.2: Аутентификация на основе подписанных cookies
* Реализован POST эндпоинт `/login`, проверяющий логин и пароль.
* При успешном входе устанавливается защищенная `httponly` cookie `session_token`.
* Токен состоит из: `user_id.timestamp.signature`. Подпись генерируется с использованием `SECRET_KEY` и хеширования.
* Реализован защищенный GET эндпоинт `/profile`, который читает cookie, проверяет подпись и валидность сессии.

### Задание 5.3: Динамическое время жизни сессии (Sliding Session)
* Реализован алгоритм "скользящей сессии".
* Максимальное время жизни сессии: 5 минут.
* При запросе к защищенному маршруту (например, `/profile`):
  * Если с момента последней активности прошло от 3 до 5 минут — сессия продлевается (кука перезаписывается новым timestamp и подписью).
  * Если прошло меньше 3 минут — кука не обновляется (оптимизация).
  * Если прошло больше 5 минут — возвращается `401 Session expired`.
* При попытке подделать `user_id`, `timestamp` или `signature` — возвращается `401 Invalid session token`.

### Задание 5.4 и 5.5: Работа с заголовками и Pydantic моделями
* Создана модель `CommonHeaders` с использованием `pydantic.Field(alias=...)` и `ConfigDict(populate_by_name=True)` для удобной работы с заголовками `User-Agent` и `Accept-Language`.
* Создана зависимость `get_common_headers`, которая инжектируется в эндпоинты через `Depends()`.
* Реализован GET эндпоинт `/headers`, возвращающий извлеченные заголовки.
* В случае отсутствия обязательных заголовков выбрасывается `HTTPException` со статусом `400 Bad Request`.
